
- If chain creation fails → raise RuntimeError (fatal — service must not start). Per user decision.
- Idempotency: If `chain upload` and `chain download` already exist in ruleset, return early.

**2. add_device_counter(mac) — ACCT-02, ACCT-03, ACCT-04**

Add counter rules to BOTH chains for the given MAC.

Upload chain rule (ACCT-02):
```
nft add rule bridge accounting upload iifname "eth1" ether saddr {mac} counter comment "bw_{mac}_up"
```

Download chain rule (ACCT-03):
```
nft add rule bridge accounting download oifname "eth1" ether daddr {mac} counter comment "bw_{mac}_down"
```

Idempotency: Check both chains for existing comment before adding.
Error handling per user decision: If upload rule succeeds but download fails, raise exception (do NOT silently leave half-added state). The caller decides how to handle.

Note: Use eth1 hardcoded for LAN interface — this is the bridge port name that doesn't change. Do NOT use _get_wan_bridge_port() (that's for WAN). eth1 is the LAN-facing interface in bridge topology.

**3. remove_device_counter(mac) — ACCT-05**

- List download chain with `-a` flag, find rules matching `bw_{mac}_down` comment, delete by handle.
- Per user decision: If MAC not found in a chain, log warning but do NOT raise exception (idempotent-close behavior).

- Merge results into single dict per MAC.
- Per user decision: On nftables read error, return empty dict `{}` and log error (bandwidth_monitor skips one cycle).

  <action>
Update bandwidth_monitor.py to work correctly with the new nft-reset counter semantics from read_device_counters().

**Key semantic change:** Previously, read_device_counters() returned CUMULATIVE byte counts (counters grow forever until chain is recreated). The bandwidth monitor calculated deltas by subtracting previous values. Now with `nft reset`, each call returns the DELTA since last read (counters reset to 0 after each read). This changes the delta calculation logic.

**Changes to _calculate_and_store_bandwidth() (lines ~86-179):**

1. **Remove delta subtraction logic.** Currently:
   ```python
   delta_up_bytes = max(0, current["upload_bytes"] - prev["upload_bytes"])
   ```
   With nft reset, the returned values ARE the delta. Change to:
   ```python
   delta_up_bytes = current["upload_bytes"]
   delta_down_bytes = current["download_bytes"]
   delta_up_pkts = current["upload_packets"]
   delta_down_pkts = current["download_packets"]
   ```

2. **Update cumulative total tracking.** For Redis `upload_total` and `download_total`, we now need to ACCUMULATE deltas ourselves:
   ```python
   # Maintain running totals in a module-level dict
   _cumulative_totals: Dict[str, Dict[str, int]] = {}
   ```
   On each cycle, add deltas to the running total. Store running total in Redis `upload_total`/`download_total` (not the raw counter value, which is now a delta).

3. **Remove _previous_counters global.** With nft-reset, there's no need to track previous counter values for delta calculation. The `_previous_counters` global and its initialization (`initial_counters = await nft.read_device_counters()` / `_previous_counters = initial_counters`) become unnecessary.

   IMPORTANT: The first `read_device_counters()` call after startup will reset counters — this is the "baseline read" that clears any stale counter values from before the service started. The bandwidth monitor should still do this first read but can discard the result (or just let the first cycle produce a slightly inflated delta — acceptable per user noting "counter deger koruma" is deferred to v2).

4. **Simplify the initial read section (lines ~269-272):**
   ```python
   # Baseline read — clears stale counters from previous service run
   await nft.read_device_counters()
   logger.info(f"Bandwidth monitor AKTIF - {POLL_INTERVAL}s aralikla bridge counter okuma (nft reset)")
   ```
   Remove `_previous_counters = initial_counters` assignment.

5. **Update the "prev is None" skip logic.** Currently:
   ```python
   prev = _previous_counters.get(mac)
   if prev is None:
       continue  # Skip first reading
   ```
   With nft-reset, every read is already a delta. Remove this skip logic — process all MACs on every cycle.

6. **Update _previous_counters references in _write_hourly_snapshot if present.** Check if that function also references _previous_counters or cumulative totals, and update accordingly.
